نصب و راه‌اندازی Installation & Setup

SignageCMS یک سیستم مدیریت نمایش دیجیتال (Digital Signage) مبتنی بر PHP 8.2 و Docker است. در عرض ۵ دقیقه راه‌اندازی کنید.

نصب Installation

1

دانلود و extract پروژه

cd C:\duc
Expand-Archive signage-cms-final.zip -DestinationPath . -Force
2

اجرای Docker

cd signage-cms
docker compose up -d --build

اولین بار ۳-۵ دقیقه طول می‌کشد.

فعال‌سازی صفحه Screen Pairing (Cookie-Based)

همه صفحات‌نمایش از یک آدرس ثابت استفاده می‌کنند. اتصال با کد فعال‌سازی یک‌بار انجام می‌شود و با کوکی ۱ ساله حفظ می‌شود.

✅ آدرس ثابت برای همه دستگاه‌ها: http://[IP-SERVER]/player/ — بدون نیاز به URL جداگانه برای هر صفحه

مراحل فعال‌سازی

1

پنل مدیریت → صفحات → انتخاب صفحه → تب «فعال‌سازی»

روی «کد جدید» کلیک کنید — یک کد ۶ کاراکتری با اعتبار ۲۴ ساعت صادر می‌شود

2

باز کردن آدرس پلیر در TV / مرورگر

در TV یا مرورگر این آدرس را باز کنید:

http://[IP-SERVER]/player/
3

وارد کردن کد فعال‌سازی

کد ۶ کاراکتری را در فیلد وارد کنید و Enter بزنید

4

✓ اتصال برقرار شد

کوکی signage_scr با مدت ۱ سال ذخیره می‌شود. دفعات بعد پلیر مستقیم باز می‌شود.

ماژول FIDS Flight Information Display System

دریافت خودکار اطلاعات پرواز از fids.airport.ir و نمایش روی صفحات. برای فرودگاه‌ها، هتل‌ها و مراکز عمومی.

راه‌اندازی اولیه

1

ماژول‌ها → FIDS → تنظیمات

فرودگاه، نوع پرواز (ورود/خروج/همه)، و حداکثر تعداد را تنظیم کنید

2

دکمه «دریافت الآن» را بزنید

اطلاعات از fids.airport.ir دریافت و در دیتابیس ذخیره می‌شود

3

Cron Job تنظیم کنید

توکن Cron را تولید کنید و دستور crontab را روی سرور اجرا کنید

دریافت خودکار با Cron

# هر ۵ دقیقه
*/5 * * * * curl -s "http://your-server/api/v1/fids/cron-sync?token=YOUR_TOKEN" > /dev/null 2>&1

رفع مشکلات Troubleshooting

راهنمای حل سریع رایج‌ترین مشکلات

مشکلات نصب

❌ Docker containers راه‌اندازی نمی‌شوند ▼
docker ps -a           # بررسی وضعیت
docker logs signage_php  # بررسی خطا
راه‌حل‌های رایج:
  • پورت 80 توسط نرم‌افزار دیگر اشغال شده → در .env پورت را تغییر دهید
  • RAM کافی نیست → Docker Desktop را به حداقل 4GB تنظیم کنید
  • docker compose down -v && docker compose up -d --build
❌ صفحه 500 Internal Server Error ▼
docker logs signage_php --tail 20
دلایل رایج:
  • Migration اجرا نشده → مراحل migration را دوباره اجرا کنید
  • PHP syntax error → لاگ را بررسی کنید

مشکلات فعال‌سازی TV

❌ «ارتباط قطع است» در TV ▼

علت: TV از IP سرور دسترسی ندارد.

  1. IP سرور را در cmd بررسی کنید: ipconfig
  2. آدرس پلیر را با IP واقعی باز کنید: http://192.168.1.10/player/SCR001
  3. TV و سرور در یک شبکه باشند
  4. Firewall ویندوز پورت 80 را باز کنید
# باز کردن پورت 80 در PowerShell (Admin)
netsh advfirewall firewall add rule name="SignageCMS" dir=in action=allow protocol=TCP localport=80
❌ «کد نامعتبر یا منقضی شده» ▼
  • کد اعتبار ۲۴ ساعت دارد — اگه گذشته از پنل «کد جدید» بگیرید
  • کد را با کپی/پیست وارد کنید تا اشتباه تایپی نباشد

مشکلات Player

❌ ویدیو پخش نمی‌شود ▼
برای Android TV / LG / Samsung:
  • پروفایل پلیر را به Android TV یا LG/Samsung تغییر دهید
  • فرمت MP4 (H.264) بیشترین سازگاری را دارد
  • WebM ممکن است در TV قدیمی کار نکند
برای همه:
  • URL فایل را بررسی کنید: http://192.168.1.10/uploads/...
  • نه localhost — باید IP واقعی باشد
❌ تصاویر لود نمی‌شوند (404) ▼
docker exec signage_php ls /var/www/html/public/uploads/media/1/
  • اگه فایل وجود دارد اما 404 می‌گیرد: uploads route فعال است

2

3

بررسی وضعیت

دکمه «بررسی وضعیت» — وضعیت GPS، تعداد ماهواره، و ADS-B را نشان می‌دهد.

4

همگام‌سازی

«همگام GPS» — یک‌بار داده را می‌خواند و تله‌متری پرواز را به‌روز می‌کند.

یا «همگام‌سازی خودکار» را فعال کنید تا هر ۱۰ ثانیه خودکار اجرا شود.

5

Push خودکار از RPi (اختیاری)

دکمه «تنظیم Push» → URL سرور، API token، و بازه زمانی را وارد کنید.

RPi مستقیماً داده را به SignageCMS می‌فرستد — بدون نیاز به باز بودن مرورگر.
