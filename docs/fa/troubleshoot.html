مشکلات نصب Docker

❌ Container ها start نمی‌شوند

docker ps -a
docker logs signage_php --tail 20
docker logs signage_mysql --tail 20
  • پورت 80 اشغال است → در .env پورت را عوض کنید: APP_PORT=8081
  • RAM کافی نیست → Docker Desktop → Settings → Resources → 4GB+
  • راه‌حل کامل: 
    docker compose down -v
docker compose up -d --build

❌ خطای 500 بعد از نصب

docker logs signage_php --tail 30
  • Migration اجرا نشده → دوباره migration را اجرا کنید
  • فایل .env ندارد → کپی از .env.example

مشکلات TV و فعال‌سازی

❌ «ارتباط قطع است» در TV

  • IP سرور را پیدا کنید: 
    ipconfig | findstr "IPv4"
  • TV باید از همان شبکه باشد
  • آدرس با IP واقعی باز کنید: http://192.168.1.10/player/SCR001
  • Firewall پورت 80 را باز کنید:
    netsh advfirewall firewall add rule name="SignageCMS" dir=in action=allow protocol=TCP localport=80

❌ «کد نامعتبر یا منقضی شده»

  • کد اعتبار ۲۴ ساعت دارد — از پنل «کد جدید» بگیرید
  • کد را با دقت وارد کنید (بدون فاصله)
  • پروفایل پلیر صفحه را به «Android TV» یا «LG/Samsung» تغییر دهید

مشکلات پلیر

❌ ویدیو پخش نمی‌شود

  • پروفایل را به Android TV یا LG WebOS یا Samsung Tizen تغییر دهید
  • فرمت MP4 (H.264) بیشترین سازگاری دارد
  • URL فایل باید با IP سرور باشد نه localhost

❌ تصویر 404 می‌دهد

docker exec signage_php ls /var/www/html/public/uploads/media/
  • فایل آپلود شده؟ → بررسی از پنل رسانه‌ها
  • جدیدترین نسخه routes/web.php باشد

❌ WebSocket کار نمی‌کند

docker logs signage_ws --tail 5
  • باید ببینید: WebSocket Server running on ws://0.0.0.0:8080
  • WebSocket اختیاری است — heartbeat هر ۱۵ ثانیه جایگزین است
  • restart: docker restart signage_ws

❌ ماژول FIDS خطا می‌دهد

Get-Content "database\migrations\003_module_tables_only.sql" -Raw -Encoding UTF8 | docker exec -i signage_mysql mysql -u signage_user -pStrongPassword123! signage_cms
